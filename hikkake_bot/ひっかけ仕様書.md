# SVML ひっかけ管理システム仕様書
**最終更新**: 2025年8月5日  
**バージョン**: 3.0  

## 概要
SVML Discord Botのひっかけ管理システムは、React風のコンポーネント分離アーキテクチャを採用し、店内状況管理から釣り客追跡まで包括的な店舗運営支援を提供します。

---

## 👥 スタッフ数管理

### スタッフ数入力フロー
1. **ボタンクリック**: プラカマスタッフ数入力
2. **プラ選択**: リスト選択（0-24）
3. **カマ選択**: リスト選択（0-24）
4. **自動更新**: 店内状況とひっかけ一覧の即座更新
5. **ログ出力**: スレッドとチャンネルへの自動ログ
6. **出退勤連携**: 完成後は`syuttaikin_bot`との自動連携予定

### ログ出力形式
**スレッドログ**: `年月日-ひっかけ`（既存がある場合は同スレッドに追記）  
**チャンネルログ**: 
```
スタッフ数を変更しました。
プラ 0→X カマ 0→X
変更時間：HH:MM 変更者：@ユーザー
スレッドリンク
```

## 🎣 ひっかけ管理プロセス

### ひっかけ予定登録
1. **ボタンクリック**: ひっかけ予定
2. **段階選択**:
   - **客数**: 1-24リスト選択
   - **プラ減少**: 0-24リスト選択
   - **カマ減少**: 0-24リスト選択
   - **本数**: 0-24リスト選択
3. **自動計算**: プラカマスタッフ数の自動減算
4. **パネル更新**: 店内状況・ひっかけ一覧の同時更新
5. **通知**: チャンネルとスレッドへの自動通知

### ひっかけ確定処理
1. **ボタンクリック**: ひっかけ確定
2. **予定選択**: 既存ひっかけ予定からリスト選択
3. **ステータス変更**: 「予定」→「確定」への自動変更
4. **ログ連携**: 元の予定ログとの自動リンク
5. **通知**: `🎣 ひっかけ確定！`の自動通知

### ひっかけ失敗処理
1. **ボタンクリック**: ひっかけ失敗
2. **予定選択**: 既存ひっかけ予定からリスト選択
3. **ステータス削除**: 予定の完全削除
4. **スタッフ復帰**: 減少していたプラカマの自動復帰
5. **励ましメッセージ**: `どんまい！次ある！`の自動送信

### 退店処理
1. **ボタンクリック**: 退店
2. **対象選択**: ひっかけ一覧からリスト選択
3. **完了記録**: 「確定」→「退店」への状態変更
4. **売上記録**: 確定本数の売上反映
5. **お疲れ様通知**: `🏠 お帰りです。ひっかけ確定人数：X`

## 🔗 出退勤Bot連携

### 自動連携機能（開発予定）
- **データソース**: `data-svml/<GUILD_ID>/syuttaikin/日付_出退勤.csv`
- **リアルタイム更新**: キャスト出退勤時のプラカマ自動反映
- **同期処理**: 出勤・退勤通知の即座反映

### 連携フロー
1. **出勤通知**: `syuttaikin_bot`からの出勤イベント受信
2. **自動更新**: プラカマスタッフ数の自動増加
3. **パネル更新**: 店内状況の即座反映
4. **退勤処理**: 退勤時の自動減算

## 📁 データ構造（GCS）

### 基本データ構造
```
data-svml/<GUILD_ID>/hikkake/
├── config.json                     # ひっかけ設定
├── stores.json                     # 店舗情報
├── daily/                          # 日次データ
│   ├── <日付>_store_status.json   # 店内状況
│   ├── <日付>_hikkake_list.json   # ひっかけ一覧
│   └── <日付>_staff_changes.json  # スタッフ変更履歴
├── hikkake_records/                 # ひっかけ記録
│   ├── scheduled.json              # 予定中
│   ├── confirmed.json              # 確定済み
│   ├── failed.json                 # 失敗記録
│   └── completed.json              # 完了記録
└── logs/                           # ログデータ
    ├── action_logs.json            # アクションログ
    └── calculation_logs.json       # 計算ログ
```

### JSON形式例
```json
{
  "hikkake_id": "hk_20250115_001",
  "store_name": "star",
  "status": "scheduled",
  "customer_count": 5,
  "staff_reduction": {
    "pla": 2,
    "kama": 1
  },
  "bottle_count": 3,
  "input_time": "2025-01-15T20:30:00Z",
  "input_user": "USER_ID",
  "thread_link": "https://discord.com/...",
  "channel_link": "https://discord.com/..."
}
```

## 🔄 自動メッセージ更新

### メッセージ更新戦略
- **削除再設置**: 古いメッセージ削除→新メッセージ設置
- **Embed更新**: 差分更新によるパフォーマンス最適化
- **ボタン保持**: ユーザー体験を損なわない更新

### 更新タイミング
- **スタッフ数変更時**: 店内状況の即座更新
- **ひっかけ状態変更時**: ひっかけ一覧の即座更新
- **出退勤連携時**: リアルタイム反映
