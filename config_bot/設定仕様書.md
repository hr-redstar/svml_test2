# SVML 設定システム仕様書
**最終更新**: 2025年8月7日  
**バージョン**: 4.0  

## 概要
SVML Discord Botの設定システムは、React風のコンポーネント分離アーキテクチャを採用し、店舗管理、ユーザー情報管理、パネル自動更新、各種Bot機能の統合設定を提供します。高速キャッシュシステムとリアルタイム更新機能により、スムーズな設定体験を実現しています。

---

## 🎛️ メインコマンド

### `/svml設定`
- **権限**: 管理者のみ
- **表示**: 全員に見える形で設定パネルを表示
- **機能**: 統合設定パネルを表示し、各種設定へのアクセスを提供

## 📊 設定パネル構成

### メイン設定パネル
**タイトル**: `⚙️ SVML設定`  
**説明**: 設定項目をボタンで選択

#### 表示情報（リアルタイム更新）
- **ログチャンネルスレッド一覧**: 動的に実際のスレッドIDを表示
  - 📋 設定ログ: `<#スレッドID>` または 未設定
  - ⚡ コマンドログ: `<#スレッドID>` または 未設定  
  - 🌍 グローバルログ: `<#スレッドID>` または 未設定
- **登録済み店舗一覧**: コピー可能なコードブロック形式
- **登録済み役職一覧**: コピー可能なコードブロック形式

#### アクションボタン（2行配置）
**1行目:**
1. **店舗名設定** (`core_store_config_button`)
2. **役職設定** (`core_role_config_button`)

**2行目:**
3. **ログチャンネル設定** (`core_log_channel_button`)
4. **ユーザー情報登録** (`core_user_info_config_button`) ✨NEW

#### パネル自動更新機能
- **即座更新**: ボタン操作後に設定パネルが自動的に最新状態に更新
- **バッチ更新**: 2秒間のバッファでまとめて更新処理
- **更新通知**: 一時的な更新完了メッセージ（5秒後に自動削除）
- **パネル情報永続化**: メッセージIDとチャンネルIDを保存して継続的な更新を実現

## 🏪 店舗名設定機能

### 設定フロー
1. **ボタンクリック**: 店舗名設定ボタン
2. **モーダル表示**: 店舗名入力モーダル
3. **入力形式**: 改行区切りで複数店舗対応
   ```
   star
   マンマミーア
   新店舗名
   ```
4. **保存処理**: GCSに店舗名データを保存
5. **自動ログ**: 設定変更ログスレッドに記録
6. **パネル更新**: 
   - 即座更新: ユーザー応答削除後に設定パネルを即座に更新
   - バッチ更新: 2秒後にキューからバッチ処理で再更新
7. **更新通知**: 一時的な完了通知（5秒後に自動削除）

### ログ形式
- **スレッド**: `設定変更ログ`スレッド
- **Embed情報**:
  - 実行者情報
  - 追加された店舗名
  - 削除された店舗名  
  - 現在の全店舗名一覧
  - タイムスタンプ

## 👥 ユーザー情報登録機能 ✨NEW

### 設定フロー
1. **ボタンクリック**: ユーザー情報登録ボタン
2. **ユーザー選択**: ユーザー選択メニューを表示
   - サーバー内の全ユーザーから選択可能
   - 自分以外のユーザーも選択可能（管理者権限）
3. **モーダル表示**: 選択されたユーザーの情報入力フォーム
   - **所属店舗**: テキスト入力
   - **役職**: テキスト入力  
   - **ニックネーム**: テキスト入力（任意）
   - **備考**: テキスト入力（任意）
4. **既存情報読み込み**: 既にユーザー情報が登録済みの場合、フォームに事前入力
5. **保存処理**: GCSにユーザー情報を保存
6. **自動ログ**: 設定変更ログスレッドに記録
7. **完了通知**: 登録完了メッセージを表示

### データ構造
```json
{
  "storeName": "店舗名",
  "role": "役職名", 
  "nickname": "ニックネーム（任意）",
  "notes": "備考（任意）"
}
```

### ログ形式
- **スレッド**: `設定変更ログ`スレッド
- **Embed情報**:
  - 対象ユーザー情報
  - 登録者情報
  - 登録内容（店舗、役職、ニックネーム、備考）
  - タイムスタンプ

## 📝 ログチャンネル設定機能

### 設定フロー
1. **ボタンクリック**: ログチャンネル設定ボタン
2. **チャンネル選択**: チャンネル選択メニューを表示
3. **自動スレッド作成**: 選択されたチャンネルに3つのスレッドを自動作成
   - **設定変更ログ**: 設定変更の履歴
   - **コマンド実行ログ**: コマンド実行の履歴  
   - **システムログ**: システム関連のログ
4. **設定保存**: ログチャンネルIDとスレッドIDsを保存
5. **完了通知**: 設定完了とスレッド一覧を表示
6. **パネル更新**: ログスレッド情報をリアルタイムで設定パネルに反映

## 🔧 統合機能

### パネル自動更新システム ✨NEW
**即座更新機能:**
- 設定変更後に`updateMainPanelImmediately()`で即座にパネル更新
- ユーザーの個別応答は削除して見た目をスッキリ
- 設定パネルの情報をリアルタイムで最新状態に反映

**バッチ更新システム:**
- `panelUpdateService`による非同期バッチ処理
- 2秒間のバッファで複数の更新をまとめて処理
- パフォーマンス最適化と負荷分散

**パネル情報永続化:**
- パネルのメッセージIDとチャンネルIDを設定状態に保存
- `/svml設定`コマンド実行時に自動保存
- 継続的な自動更新を実現

### 自動ログスレッド管理
- **3種類のログスレッド**: 設定変更、コマンド実行、システム
- **自動作成**: ログチャンネル設定時に一括作成
- **動的表示**: 設定パネルでスレッドIDをリアルタイム表示
- **アクセス制御**: スレッド取得失敗時のフォールバック機能

### キャッシュシステム ✨NEW
**StateCache:**
- TTL（Time To Live）による自動期限管理
- メモリ効率化のための期限切れエントリ自動削除
- 高速なデータアクセスとGCS負荷軽減

**PerformanceManager:**
- 並列タスク実行管理
- キャッシュヒット率の追跡
- システムメトリクスの監視

## 🚀 パフォーマンス

### 高速化対策
- **並列処理**: `Promise.all()`でGCSアクセスとデータ取得を並列化
- **キャッシュシステム**: StateCache・PerformanceManagerによる高速データアクセス
- **キャッシュヒット率**: 70%以上を維持（定期監視）
- **メモリ管理**: 期限切れキャッシュの自動削除とメモリ最適化
- **バッチ処理**: パネル更新の効率的なキュー管理

### エラーハンドリング
- **段階的フォールバック**: GCS接続失敗時のデフォルト表示
- **詳細ログ**: デバッグレベルでの詳細なエラー追跡
- **ユーザー継続**: エラー時もユーザー操作継続可能
- **リトライ機能**: 一時的な障害に対する自動リトライ
- **メッセージ取得エラー**: パネル更新時のチャンネル・メッセージ取得失敗対応

### 監視・メトリクス ✨NEW
- **システムメトリクス**: メモリ使用量、CPU使用率、稼働時間
- **キャッシュ統計**: ヒット率、サイズ、クリーンアップ頻度
- **タスク実行統計**: 実行回数、平均実行時間
- **定期レポート**: 1分毎のパフォーマンス状況出力
