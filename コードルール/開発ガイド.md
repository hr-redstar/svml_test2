## Bot作成ルール

### 基本構造
```
bot_name/
├── index.js          # メインエントリーポイント
├── commands/          # スラッシュコマンド
├── components/        # ボタン・セレクト等のコンポーネント
├── handlers/          # インタラクション処理
├── utils/            # ユーティリティ関数
└── constants/        # 定数定義
```

### 必須実装項目
- [ ] `index.js` でcommands配列とcomponentHandlers配列をエクスポート
- [ ] エラーハンドリングの統一
- [ ] loggerシステムの使用（console.log禁止）
- [ ] @rootエイリアスの使用

## コマンド作成ルール

### SlashCommandBuilder基本形式
```javascript
const { SlashCommandBuilder, PermissionFlagsBits } = require('discord.js');
const logger = require('@root/common/logger');

module.exports = {
  data: new SlashCommandBuilder()
    .setName('コマンド名')
    .setDescription('コマンドの説明')
    .setDefaultMemberPermissions(PermissionFlagsBits.Administrator),

  async execute(interaction) {
    await interaction.deferReply({ ephemeral: false });
    
    try {
      // コマンド処理
      await interaction.editReply({
        content: '処理完了',
        embeds: [embed],
        components: [row]
      });
    } catch (error) {
      logger.error(`[${interaction.commandName}] エラー:`, error);
      await interaction.editReply({
        content: '❌ エラーが発生しました。'
      });
    }
  },
};
```

### 命名規則
- **コマンド名**: `/svml設定xxxx` 形式
- **ファイル名**: `commandName.js` (camelCase)
- **説明文**: 具体的で分かりやすく

## ハンドラー実装ルール

### インタラクション処理の基本形式
```javascript
const logger = require('@root/common/logger');

module.exports = {
  name: 'handlerName',
  
  async execute(interaction) {
    try {
      // customIdの解析
      const [action, ...params] = interaction.customId.split('_');
      
      switch (action) {
        case 'button1':
          await handleButton1(interaction, params);
          break;
        default:
          logger.warn(`未対応のcustomId: ${interaction.customId}`);
      }
    } catch (error) {
      logger.error('ハンドラーエラー:', error);
      
      // エラー応答
      try {
        if (!interaction.replied && !interaction.deferred) {
          await interaction.reply({
            content: '❌ 処理中にエラーが発生しました。',
            ephemeral: true
          });
        }
      } catch (replyError) {
        logger.error('エラー応答失敗:', replyError);
      }
    }
  }
};
```

### customId命名規則
- `action_param1_param2` 形式
- アンダースコアで区切り
- 32文字以内

## エラーハンドリング

### 必須エラー処理
```javascript
try {
  // メイン処理
} catch (error) {
  logger.error('処理エラー:', error);
  
  // GitHub Actions環境での特別処理
  if (process.env.GITHUB_ACTIONS) {
    logger.error('GitHub Actions環境でのエラー検出');
    return; // process.exit(1) 禁止
  }
  
  // ユーザーへのエラー通知
  try {
    await interaction.editReply({
      content: '❌ エラーが発生しました。管理者に報告してください。'
    });
  } catch (replyError) {
    logger.error('エラー応答失敗:', replyError);
  }
}
```

### エラーレベル
- `logger.error()` - システムエラー
- `logger.warn()` - 警告レベル
- `logger.info()` - 情報レベル
- `logger.debug()` - デバッグ情報

## 禁止事項

### ❌ 絶対に禁止
1. **console.log/console.error** の使用
   - 必ずloggerを使用

2. **process.exit(1)** の使用
   - GitHub Actions環境でコンテナが終了する

3. **相対パス** の使用
   - 必ず@rootエイリアスを使用

4. **ハードコーディング**
   - 環境変数や設定ファイルを使用

5. **未処理のPromise**
   - 必ずtry-catchでエラー処理

### ⚠️ 注意が必要
1. **Discord API制限**
   - レート制限の考慮
   - タイムアウト処理の実装

2. **メモリリーク**
   - 適切なリソース解放
   - キャッシュの適切な管理

3. **セキュリティ**
   - 入力値の検証
   - 権限チェック
